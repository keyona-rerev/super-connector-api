### 2. Contacts CRM — Full New Page

**Files added to `keyona-rerev/super-connector-app`:**

| File | Purpose |
|---|---|
| `contacts-crm.js` | Full Contacts CRM extension (see below) |
| `config.js` | Updated — now auto-loads `contacts-crm.js` on `window.load` |
| `load-crm.js` | Backup bootstrap loader (not strictly needed now) |

**How it loads:** `config.js` is already in `index.html`. The updated `config.js` fires `window.addEventListener('load', inject)` which appends `contacts-crm.js` to the DOM. This avoids any need to edit `index.html` directly.

**What `contacts-crm.js` adds at runtime:**
- **Nav item:** `⊞ Contacts` inserted before "Search Contacts" in the sidebar
- **`page-contacts` div:** Inserted before `page-search` — contains the toolbar, grid, and pagination
- **Contact modal:** `#contact-modal-overlay` — 12-field form for Add and Edit, appended to `<body>`
- **Edit button:** `✎ Edit Contact` button added to the contact drawer header

**Contacts page behavior (Option B from handoff):**
- Default: Browse grid of 50 contacts, paginated, `GET /contacts?limit=50&offset=N`
- Filters: Venture, Relationship Health, Activation Potential dropdowns
- Typing 1-3 chars: client-side filter of the current 50-contact page
- Typing 4+ chars or pressing Enter: fires `POST /search` vector search, shows results with match % badges
- Clear button: resets to browse mode
- `+ New Contact` in topbar (when on Contacts page): opens Add modal
- Save: `POST /contact` — vectorizes on Railway automatically
- Edit from drawer: `PUT /contact/{id}` — re-vectorizes automatically

**Contact fields in modal:** full_name*, title_role, organization, venture, **source** (new), how_we_met, relationship_health, activation_potential, what_building, what_need, what_offer, notes
