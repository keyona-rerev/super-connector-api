**URL:** https://super-connector-api-production.up.railway.app  
**Repo:** github.com/keyona-rerev/super-connector-api  
**Auth:** `X-API-Key: SC_API_KEY` header on all requests except `/health`

**Key endpoints:**
- `POST /search` — semantic search across contacts (the crown jewel)
- `GET /match/{contact_id}` — find similar contacts by vector
- `POST /draft` — auto-draft warm intro email between two contacts
- `GET /initiatives` — all initiatives
- `GET /initiative/{id}` — full initiative with sub-projects, stakeholders, action items
- `PATCH /initiative/{id}/status` — quick status update
- `POST /action-item` — create action item
- `GET /action-items?due_before=YYYY-MM-DD` — Phoebe's overdue list

## DESIGN PRINCIPLES (from Keyona)

1. **No side panels** — modals should be full-screen centered, 80-90% viewport
2. **Initiative = project container** — should show its full internal structure (sub-projects, stakeholders, contacts) in one place
3. **Information given = information captured = information referenced** — Phoebe never asks the same thing twice
4. **Semantic matching is the point** — every stakeholder profile should automatically surface matched contacts from the vector database
5. **Claude-inspired visual design** — soft colors, warm tones, not dark/aggressive aesthetics
6. **Everything hands-off** — if Keyona has to manually do something more than once, it should be automated
