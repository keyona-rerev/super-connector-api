## NEXT SESSION: EVENTS FEATURE

### Design Decision (agreed)
**Option C — Hybrid model:**
- Interview series episodes stay as **sub-projects** under their initiative (already modeled correctly)
- All other events (panels, workshops, networking) get a new **Events** first-class object

### New Railway Tables Needed

**`events` table:**
```
event_id          TEXT PRIMARY KEY  (auto: EVT-{timestamp})
event_name        TEXT
event_type        TEXT  -- 'Hosting' | 'Attending' | 'Workshop'
venture           TEXT
date              DATE
location          TEXT
status            TEXT  -- 'Planning' | 'Confirmed' | 'Complete' | 'Cancelled'
initiative_id     TEXT  (FK → initiatives, nullable)
description       TEXT
notes             TEXT
created_at        TIMESTAMP
```

**`event_guests` join table:**
```
guest_id          TEXT PRIMARY KEY  (auto: EG-{timestamp})
event_id          TEXT  (FK → events)
contact_id        TEXT  (FK → contacts, nullable — can add before contact exists)
full_name         TEXT
role              TEXT  -- 'Speaker' | 'Panelist' | 'Attendee' | 'Interviewee' | 'Organizer' | 'Sponsor'
guest_status      TEXT  -- 'Invited' | 'Confirmed' | 'Attended' | 'No-show' | 'Declined'
notes             TEXT
```

### New Railway Endpoints Needed

```
GET    /events                          List all events (?type=Hosting&venture=BTC)
POST   /event                           Create event
GET    /event/{id}                      Fetch event + guests
PUT    /event/{id}                      Update event
DELETE /event/{id}                      Remove event
PATCH  /event/{id}/status               Quick status update
GET    /event/{id}/guests               List guests for event
POST   /event-guest                     Add guest to event
PATCH  /event-guest/{id}               Update guest status/role
DELETE /event-guest/{id}               Remove guest from event
```

### Web App UI Design

**New nav item:** `◈ Events` (below Content Registry)

**Page layout:** Three tabs — **Hosting · Attending · Workshops**

**Event card shows:**
- Event name, date, status badge
- Venture tag, connected initiative (if any)
- Guest count badge (e.g. "4 confirmed / 6 invited")

**Event modal (full-screen, same pattern as initiative modal):**
- Header: name, type, date, venue, status, venture, connected initiative
- Section 1: **Guest Pipeline** — contacts from Railway with status per guest (Invited → Confirmed → Attended). Add guest by searching Railway contacts. Each guest card shows name, role, status, action chips (Follow Up, Send Recap, etc.)
- Section 2: **Connected Initiative** — links to initiative if set
- Section 3: **Post-Event Actions** — auto-generated follow-up action items, links to any content assets produced

**Interview series episodes (existing sub-project pipeline):**
- When you click a sub-project inside the initiative modal, the module detail panel should show:
  - Guest slots (each slot = a contact/stakeholder with their interview status)
  - Production status (Pitching → Scheduled → Recorded → Editing → Published)
  - Content asset output link
